## 1. 프로젝트 개요

패션 트렌드가 **런웨이 → 전문가 → 셀럽 → 마켓 → 다음 시즌**으로 전파되는 과정을 데이터 기반으로 추적·분석하는 대시보드.

**핵심 가치**: F&F AX팀이 트렌드 전파의 타이밍과 강도를 정량적으로 파악하여, 상품 기획·바잉 의사결정에 데이터 근거를 제공한다.

**포지셔닝**: "감을 대체하는 도구"가 아니라, **감의 근거를 제공하고 블라인드 스팟을 잡아주는 도구.**

**방법론**: "기능을 만들고 맞는지 확인"이 아니라, **"방법론을 검증하고 검증된 것만 구현".**

## 3. 핵심 가설 & 검증 프레임워크

### 대전제 가설

> "패션 트렌드는 다양한 기원(Origin)에서 출발하여 각기 다른 경로로 전파되며, 그 기원과 경로에 따라 전파 속도·강도·지속성이 다르다."

### 하위 가설

| # | 가설 | 검증 방법 |
|---|------|----------|
| H1 | 런웨이-마켓 간 유의미한 상관관계 존재 | 런웨이 Top 30 vs 마켓 전수 매칭 |
| H2 | 전문가 리포트에 독립적 예측력 존재 | 풀 A(교집합) vs 풀 B(전문가 독자) 적중률 비교 |
| H3 | 셀럽 착용은 마켓 확산의 선행 지표 | 검색량 스파이크 → 마켓 상품 수 시차 분석 |
| H4 | 시즌 간 변화로 트렌드 방향 판별 가능 | 24SS→26FW 태그 변화 + VLM 시각 속성 비교 |
| H5 | Origin 타입에 따라 전파 패턴이 구분 가능 | 시그널 발생 순서 분석으로 자동 분류 |

### "적중" 정의 (3단계)

| 레벨 | 기준 | 측정 방법 | 구현 |
|------|------|----------|------|
| L1 검색 관심 | 네이버 DataLab 주간 검색량 +30% | Naver DataLab API | 우선 |
| L2 마켓 반영 | 키워드 포함 마켓 상품 수 증가 | 반복 크롤링 + 매칭 | 다음 |
| L3 판매 검증 | 품절률·판매 순위 상승 | 크롤러 재고 상태 추가 | 이후 |

### 트렌드 기원(Origin) 4가지 분류

각 Origin은 **서로 다른 전파 경로**를 가진다. 같은 파이프라인의 다른 경로가 아니라, **아예 다른 파이프라인.**

| 타입 | 시그널 플로우 | 특징 |
|------|-------------|------|
| **Runway-led** | 런웨이 → 전문가 → 셀럽 → 검색 → 마켓 | 순차적 전파, 예측 가능, 딜레이 6~12개월 |
| **Capital-driven** | 브랜드 투자 → 앰배서더 캠페인 → 검색 폭발 → 마켓 | 런웨이/전문가 건너뜀, 캠페인 시점에 급등 |
| **Viral/Meme** | 소셜 밈 발생 → 틱톡/릴스 확산 → 검색 급등 → 마켓 빠른 소진 | 모든 전통 채널 우회, 빠르지만 단명 |
| **Market-organic** | 소비자 실수요 → 마켓 점진적 확대 → 검색 완만 상승 | 선행 시그널 없이 점진적, 지속적 성장 |

**Origin 자동 분류 핵심**: "어떤 시그널이 먼저 나타났는가"로 판별.

### 패션 조닝별 Origin 지배 구조

| 조닝 | Runway-led | Capital-driven | Viral/Meme | Market-organic |
|------|-----------|----------------|------------|----------------|
| **럭셔리/하이엔드** | **60%** | 25% | 10% | 5% |
| **스포츠/아웃도어** | 15% | 30% | 15% | **40%** |
| **캐주얼/스트리트** | 20% | 30% | **35%** | 15% |
| **SPA/매스** | 25% | 20% | 30% | 25% |

**조닝에 따라 어떤 Origin이 지배적인지가 다르므로, 모니터링 우선순위도 달라야 한다.**

## 6. Trend Flow 탭 — Origin 기반 재설계 ⭐

### 설계 원칙

기존 5단계 선형 파이프라인은 Runway-led 하나만 설명하는 구조.
**v4에서는 "키워드 중심 뷰"로 전환** — 키워드마다 어떤 시그널이 먼저 나타났는지에 따라 Origin을 자동 분류하고, Origin별로 다른 플로우를 시각화한다.

### Origin별 시그널 플로우 상세

**Runway-led 플로우:**
```
[런웨이 컬렉션] → [전문가 리포트] → [셀럽 착용] → [마켓 등장]
     활성              활성              활성           활성
                                   ↘
                              [검색량 상승]
                                   활성

(소셜 멘션) (캠페인 런칭) ← 비활성 (점선)
T+0          T+1~2M        T+3~6M       T+6~12M
```

**Capital-driven 플로우:**
```
(런웨이) (전문가) ← 비활성 (점선)

[브랜드 투자 결정] → [앰배서더 캠페인] → [마켓 빠른 반영]
      활성                 활성               활성
                      ↘
                 [검색량 폭발]
                      활성

캠페인 시점            T+0         T+1~3M
```

**Viral/Meme 플로우:**
```
(런웨이) (전문가) (셀럽 착용) ← 전부 비활성

[소셜 밈 발생] → [틱톡/릴스 확산] → [검색량 급등] → [마켓 빠른 소진]
     활성              활성             활성            활성

밈 발생!          확산 1~2주        T+2~4주        T+1~3M
```

**Market-organic 플로우:**
```
(런웨이) (전문가) (셀럽 착용) ← 전부 비활성

[소비자 실수요] → [마켓 점진적 확대] → [검색량 완만 상승]
     활성               활성                활성

선행 시그널 없음       점진적 증가         지속 성장
```

### Origin 자동 분류 로직

```python
def classify_origin(keyword_signals: dict) -> str:
    """
    시그널 발생 순서로 Origin 자동 분류.
    keyword_signals = {
        'runway_first': '24FW',      # 런웨이 첫 등장 시즌
        'expert_first': '25SS',      # 전문가 첫 언급
        'celeb_first': '25FW',       # 셀럽 첫 감지
        'search_first': '25SS',      # 검색량 첫 스파이크
        'social_first': None,        # 소셜 멘션 첫 감지
        'market_first': '25FW',      # 마켓 첫 등장
        'campaign_detected': False,  # 앰배서더 캠페인 여부
    }
    """
    s = keyword_signals
    
    # Runway-led: 런웨이가 가장 먼저, 순차적 전파
    if s['runway_first'] and is_earliest(s['runway_first'], s):
        return 'runway_led'
    
    # Capital-driven: 캠페인 감지 + 셀럽+검색 동시 급등
    if s['campaign_detected'] and s['celeb_first'] and not s['runway_first']:
        return 'capital_driven'
    
    # Viral/Meme: 소셜이 먼저, 런웨이/전문가 부재
    if s['social_first'] and is_earliest(s['social_first'], s) and not s['runway_first']:
        return 'viral_meme'
    
    # Market-organic: 마켓이 먼저, 선행 시그널 부재
    if s['market_first'] and not s['runway_first'] and not s['social_first']:
        return 'market_organic'
    
    return 'unknown'
```

### Brand Board 연결

- Trend Flow에서 키워드 클릭 → Brand Board가 해당 키워드로 필터링된 상품 그리드 표시
- Brand Board에서 상품 브라우징 → Trend Flow로 역추적 링크
- 조닝 필터 연동: Trend Flow에서 "캐주얼" 선택 시 Brand Board도 캐주얼 브랜드만 표시

## 7. 백테스트 프레임워크

### Step 1: 시그널 수집
- 24SS~25FW 런웨이 태그 (Tagwalk 크롤링)
- 전문가 리포트 키워드 (WGSN, Tagwalk, Pantone 등)
- → 풀 A/B 분류

### Step 2: "뭐가 핫했는가" 정의

**객관 데이터:** 무신사 랭킹, 네이버 검색량(보유), 네이버 쇼핑 인사이트, Kream 프리미엄
**수기 회고:** MD/디자이너 인터뷰 → hot_item_log (evidence_type 명시)

### Step 3: 매칭 & 시차 분석
- 풀 A/B vs 핫 아이템 적중률
- 키워드별 "첫 시그널 → 마켓 확산" 딜레이
- 불발 키워드 별도 분석

### Step 4: Origin 역추적
- 핫 아이템 10~15개에 시간순서 + 시그널 유무 기반 Origin 분류
- MD 독립 평가와 교차 검증

### Step 5: 방법론 평가
- Origin 타입별 FTIB 포착률 → 잘 잡는 영역 / 보완 필요 영역 명확화
- 풀 A vs 풀 B 적중률 → 전문가 독립 예측력 판단
